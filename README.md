# momoAI — USDC Payment Assistant on Arc

**AI-written, permanently on-chain payment memos for USDC transfers on Arc Testnet.**

momoAI lets you send USDC with a clear, professional memo attached to every transaction — generated by AI and recorded permanently on-chain through Arc's native Memo contract. No off-chain database, no lost context.

🔗 **Live app:** [arc-memo-ai.vercel.app](https://arc-memo-ai.vercel.app)
💻 **Repo:** [github.com/Zaksonboy/MEMOAI-ON-ARC](https://github.com/Zaksonboy/MEMOAI-ON-ARC)

---

## What it does

1. Connect your wallet — the app detects and switches to Arc Testnet automatically.
2. Enter a recipient address, an amount, and a short description of what the payment is for.
3. Tap **Generate** — AI turns your description into a clean, professional memo.
4. The app builds a full structured record: sender, recipient, amount, reason, date, and time.
5. Tap **Send Payment** — the app calls Arc's Memo contract, which wraps a real USDC transfer and attaches the memo as part of the same on-chain transaction.
6. Get a direct link to the confirmed transaction on Arc's explorer, and see it saved in your local payment history.

## Why it matters

Standard token transfers only show an amount and two addresses — no context. momoAI attaches the *reason* for a payment directly to the transaction itself, using Arc's purpose-built Memo contract, so the explanation is permanent, public, and doesn't depend on any external database staying online.

| Backend | Vercel serverless functions |
| Hosting | Vercel |

## Contracts used

momoAI doesn't deploy its own contract — it calls two contracts already provided by Arc:

- **USDC (system contract):** `0x3600000000000000000000000000000000000000`
- **Memo contract:** `0x5294E9927c3306DcBaDb03fe70b92e01cCede505`

The Memo contract wraps a standard USDC `transfer()` call and attaches your memo in the same transaction, while preserving your wallet as the true sender.

## Running it yourself

1. Clone the repo
2. Set the following environment variable in your Vercel project settings:
   - `GEMINI_API_KEY` — a free API key from [Google AI Studio](https://aistudio.google.com)
3. Deploy to Vercel — no build step required, it's a static frontend with one serverless function

## A note on how this was built

Every part of momoAI — the frontend, the serverless backend, the smart contract integration, and every bug fix along the way — was built and deployed entirely from a mobile phone, using GitHub's mobile editor, Vercel's dashboard, and a wallet app's built-in browser for testing. No laptop, no terminal.
